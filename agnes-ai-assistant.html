🔑 第一步：创建 API Key
  1. 访问 Agnes AI 控制台 并注册/登录账户
  2. 进入 Settings → API Keys
  3. 点击 Create new secret key
  4. 复制并妥善保存（只显示一次）
⚠️ 安全提醒：API Key 是敏感信息，请勿暴露在代码仓库、前端代码、截图或公开文档中。如泄露请立即在控制台重置。

📡 第三步：配置请求地址与认证

所有请求统一使用以下 Base URL 和认证头：

Base URL: https://apihub.agnes-ai.com/v1

Headers:
  Authorization: Bearer YOUR_API_KEY
  Content-Type: application/json
💡 迁移提示：如果你之前使用 OpenAI API，只需改 3 处：Base URL → Agnes 的地址，API Key → Agnes 的 Key，model 名称 → Agnes 的模型名。

💡 Thinking 模式详解

Thinking 模式让模型在正式回答前先进行内部推理，显著提升编程、调试、推理和 Agent 任务的质量。
OpenAI 兼容格式

在请求体中添加 chat_template_kwargs.enable_thinking

{
  "model": "agnes-2.0-flash",
  "messages": [...],
  "chat_template_kwargs": {
    "enable_thinking": true
  }
}
Anthropic 兼容格式

使用 thinking 字段控制预算

{
  "model": "agnes-2.0-flash",
  "messages": [...],
  "thinking": {
    "type": "enabled",
    "budget_tokens": 2048
  }
}
💡 budget_tokens 建议从 2048 开始，复杂任务可适当提高
🎯 何时开启 Thinking？
场景建议budget_tokens
简单问答 / 闲聊不需要-
代码生成 / 调试强烈建议2048
复杂推理 / 数学强烈建议4096+
多步骤 Agent 任务强烈建议4096+
代码重构 / 架构设计强烈建议2048-4096

📝 提示词最佳实践

文生视频结构

[主体] + [动作] + [场景] + [镜头运动] + [光照] + [风格]

例：A young astronaut walking across a red desert planet, dust blowing in the wind, slow cinematic tracking shot, dramatic sunset lighting, realistic sci-fi style

❓ 常见问题 FAQ

🔐 认证与账户

401 Unauthorized 怎么办？+
1. 检查 API Key 是否正确（Bearer 后有空格）
2. 确认 Key 未过期或被删除
3. 检查 Header 拼写：Authorization: Bearer YOUR_KEY
4. 确认账户有余额（注册送 $0.1）
API Key 泄露了怎么办？+
立即进入控制台 Settings → API Keys 删除旧 Key 并创建新 Key。检查代码仓库历史记录，使用 git filter-repo 或 BFG 清理敏感信息。
免费额度用完了还能用吗？+
当前所有模型均为 $0 现价，免费账户调用免费模型不扣费，但有 RPM（每分钟请求数）限制。如需更高并发，请充值。

💬 文本模型

如何开启 Thinking 模式？+
OpenAI 格式加 chat_template_kwargs.enable_thinking: true；Anthropic 格式加 thinking: {type:"enabled", budget_tokens:2048}。详见左侧「Thinking 模式」页面。
图片 URL 输入报错怎么办？+
1. 确保图片 URL 公网可访问（无需登录/Cookie）
2. 使用标准格式：JPG、PNG、WebP
3. 在 messages[].content 中使用数组格式，包含 {type:"image_url", image_url:{url:"..."}}
4. 防盗链图片可能无法读取
流式输出如何接入？+
在请求体中加 "stream": true。响应会逐块返回 SSE 格式数据，每块以 data: 开头，最后以 data: [DONE] 结束。
工具调用返回格式不对？+
确保 tools 数组格式正确，包含 type、function.name、function.description、function.parameters。模型可能返回 finish_reason: "tool_calls"，此时需要解析 tool_calls 数组并执行对应函数。

🖼️ 图像模型

response_format 放错位置报 400？+
不要把 response_format 放在请求体根级。正确做法：放在 extra_body 中：{"extra_body":{"response_format":"url"}}。文生图 Base64 可用 return_base64: true。
图生图需要传 tags: ["img2img"] 吗？+
不需要。当前接口已简化，图生图只需在 extra_body.image 数组中提供输入图片 URL 即可。注意：image 必须放在 extra_body 中，不能放请求体顶层！放 tags 反而可能报错。
输入图片 URL 不可访问怎么办？+
1. 换用公网 HTTPS 地址
2. 或使用 Data URI Base64 格式：data:image/png;base64,BASE64_HERE
3. 检查图片是否需登录/有防盗链
图像生成超时？+
图像生成通常需要数秒到几十秒。建议客户端超时设置为 60s - 360s。复杂提示词、大尺寸图片或服务高负载时会延长。
2.0 和 2.1 图像模型怎么选？+
新用户直接用 agnes-image-2.1-flash。2.1 在高信息密度、复杂布局、细节丰富场景表现更好，且支持结构保持。

🎬 视频模型

视频生成任务状态一直 queued？+
视频生成是异步任务，创建后需要轮询查询结果。建议轮询间隔 5 秒。高峰期队列可能较长，请耐心等待。如超过 10 分钟仍 queued，请联系 support。
num_frames 报参数错误？+
必须同时满足两个条件：
1. num_frames ≤ 441
2. num_frames = 8n + 1（即除以 8 余 1）
有效值：81, 121, 161, 241, 441
视频查询用 video_id 还是 task_id？+
新接入强烈推荐用 video_id 查询（/agnesapi?video_id=...）。task_id 接口仍兼容旧版本，但未来可能弃用。
如何生成更长的视频？+
两种方式：
1. 增加 num_frames（最大 441）
2. 降低 frame_rate（如从 24 降到 12）
公式：seconds = num_frames / frame_rate。最长约 36 秒（441 帧 @ 12 FPS）。
关键帧动画怎么设置？+
在 extra_body 中设置 mode: "keyframes"，并在 extra_body.image 中提供多张关键帧图片 URL。模型会在关键帧之间生成平滑过渡。

🔧 通用问题

400 Bad Request 常见原因？+
1. JSON 格式错误（缺少逗号、引号不匹配）
2. 必填参数缺失（如 model、prompt、size）
3. 参数类型错误（如 num_frames 传了字符串）
4. response_format 放错位置
5. 图片 URL 格式不正确
503 Service Unavailable 怎么办？+
服务暂时繁忙，建议：
1. 等待几秒后重试（指数退避）
2. 检查 Agnes AI 状态页或 Discord 公告
3. 如持续出现，联系 support@agnes-ai.com
如何查看我的用量和余额？+
登录 Agnes AI 控制台，进入 Billing → Balance 查看余额，Usage 页面查看详细用量统计。

🚨 错误码速查表

状态码含义常见原因解决方案
400请求无效参数错误、JSON 格式问题、必填项缺失检查请求体格式、参数类型、必填字段
401未授权API Key 错误、过期、格式不对检查 Authorization 头，确认 Key 有效
404不存在视频/任务 ID 不存在、URL 路径错误确认 ID 正确，检查端点路径
429速率限制请求过于频繁，超出 RPM 限制降低请求频率，实现退避重试
500服务器错误服务端内部异常稍后重试，如持续出现请联系支持
503服务繁忙服务负载高或维护中等待后重试，关注状态公告
🔄 推荐重试策略
// 指数退避示例
const delay = (ms) => new Promise(r => setTimeout(r, ms));

async function callWithRetry(fn, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (e) {
      if (e.status === 429 || e.status >= 500) {
        await delay(Math.pow(2, i) * 1000); // 1s, 2s, 4s
      } else {
        throw e; // 4xx 客户端错误不重试
      }
    }
  }
}
